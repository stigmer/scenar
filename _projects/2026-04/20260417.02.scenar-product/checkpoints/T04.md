# T04 Checkpoint — Chrome Primitives Extraction

**Date**: 2026-04-17
**Status**: Complete
**Commit**: Pending

## What was built

Extracted four generic chrome/annotation primitives from the Stigmer repo's `site/src/components/docs/demos/views/` into the Scenar monorepo's `@scenar/react` package. Introduced a minimal `--scenar-*` CSS token layer that makes shells render correctly in any host application.

### Theme foundation (new)
- **`packages/react/src/theme/tokens.css`** — 3 CSS custom properties (`--scenar-surface`, `--scenar-border`, `--scenar-foreground`) under `@layer scenar`, scoped to `.scenar` / `.scenar.dark`
- **`packages/react/src/theme/index.ts`** — exports `SCENAR_CLASS = "scenar"` constant
- **`@scenar/react/theme.css`** export path wired in `package.json` — consumers `import "@scenar/react/theme.css"` once

### Shell components (new)
- **`BrowserView`** (116 LOC) — Chrome-style browser chrome with tab strip, address bar, traffic lights, animated content transitions
- **`TerminalView`** (119 LOC) — macOS Terminal/iTerm2 emulator with title bar, tab bar, coloured prompt lines
- **`CodeEditorView`** (219 LOC) — VS Code-style editor with optional file explorer, activity bar, line-number gutter, highlight support
- **`PulseHighlight`** (30 LOC) — Animated border ring overlay for drawing attention to elements

### Supporting modules
- **`packages/react/src/shells/tokens.ts`** — `SHELL_HEIGHT_DEFAULT` (380), `SHELL_HEIGHT_MIN` (320), `BROWSER_SHELL_HEIGHT_DEFAULT` (420)
- **Barrel exports** in `shells/index.ts`, `highlights/index.ts`, and main `index.ts`

### Tests
- **4 test files, 13 tests** — all shallow-render tests using `@testing-library/react`
  - `BrowserView.test.tsx` — URL rendering, tab title derivation, zoom prop
  - `TerminalView.test.tsx` — cwd prefix, default cwd, custom title, default title
  - `CodeEditorView.test.tsx` — filename tab, explorer visible with fileTree, explorer hidden without fileTree, line highlighting
  - `PulseHighlight.test.tsx` — aria-hidden attribute, `--scenar-foreground` border colour

## Port deltas (transformations applied during extraction)

1. **Host-theme-token elimination** — `border-border` → `borderColor: "var(--scenar-border)"`, `bg-background` → `background: "var(--scenar-surface)"`, `border-foreground` → `borderColor: "var(--scenar-foreground, currentColor)"`. Zero Tailwind host-token classes remain in shells/highlights.
2. **Shell-height CSS variable** — `--demo-shell-height` → `--scenar-shell-height` (already produced by `DemoViewport`)
3. **Stigmer vocabulary removed** — `cwd` default: `~/stigmer-federation` → `~`; `workspaceName` default: `stigmer-federation` → `workspace`
4. **Interior chrome colours preserved** — All hardcoded Chrome/Terminal/VS Code palette colours stay literal (they mimic real software, not the host app's theme)
5. **`TerminalView` line colour map** — Extracted from inline object per render to module-level `LINE_COLORS` constant

## Design decisions

1. **Scope: primitives only** — `BrowserView`, `TerminalView`, `CodeEditorView`, `PulseHighlight` moved. `AppShell`, `ManagementShell`, `APIExchangeView`, `ComposerView`, `WidgetsSidebar`, `ResourceListPage` stay in Stigmer. The first three are structurally generic but labelled/shaped to Stigmer's console. The last three import `@stigmer/react` or `@stigmer/protos`.

2. **Minimal token layer** — Only 3 tokens ship (`--scenar-surface`, `--scenar-border`, `--scenar-foreground`) because only 3 are consumed by shipped components. Future tokens are per-component decisions, not speculative upfront API.

3. **`@layer scenar`** — All Scenar CSS rules live in a low-priority CSS layer so host application styles win without `!important`. Mirrors the `@layer stgm` / `--stgm-*` pattern from Stigmer's `@stigmer/theme`.

4. **`.scenar` container scope** — Token activation requires wrapping with `class="scenar"`. Consistent with `--scenar-cursor-stroke` and `--scenar-shell-height` conventions established in T03.

5. **`sideEffects` for CSS** — `package.json` `sideEffects` changed from `false` to `["./dist/theme.css"]` so bundlers don't tree-shake the CSS import.
