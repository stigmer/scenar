## Decisions Made

1. **Package name `scenar` (unscoped)**: Matches Prisma/Turbo pattern — the CLI gets the product name, scoped packages (`@scenar/*`) are programmatic libraries
2. **YAML uses snake_case**: Proto-native convention for YAML files; loader converts to camelCase for TS runtime
3. **Echogarden as optional peer dep**: GPL v3 license vs Apache-2.0 — user installs explicitly, CLI provides guidance
4. **Validation in CLI, not SDK**: Schema validation and semantic validation (view registry) are different concerns
5. **`runNarrate` accepts injected `TtsProvider`**: Dependency injection for testability over mocking `node:fs/promises`

## Learnings

- Vitest's `vi.mock()` for `node:` built-in modules requires `importOriginal` pattern with `vi.hoisted()` for factory-scoped variables — simpler to use real temp directories for file I/O tests
- Echogarden uses `any`-typed dynamic import pattern since it's an optional peer dep with no compile-time type guarantee
- The `encodeRawAudioToMp3` API in Echogarden wasn't directly accessible — provider uses dynamic import with `any` to avoid coupling to optional dependency types
